What This Project Does

Blastradius is a full-stack audit log for software deployments and configuration changes. Every deployment to DEV, TEST, STAGING or PRODUCTION is recorded with an automatically assigned risk level, and every configuration/metadata change (created, modified, deleted) is linked back to the deployment that caused it. Releases bundle multiple deployments together — expanding a release shows exactly which deployments are linked to it and the total metadata changes aggregated across them. Teams can bulk-import existing change logs via CSV (chunked ingestion, tolerant of malformed rows, with a persisted upload history), export the full audit log as a compliance CSV straight from the Deployments or Metadata pages, and follow a live dashboard with risk distribution, environment activity, and a chronological recent-activity timeline.

How It Works — 5-Step Flow

1

Developer logs in via JWT

POST /api/auth/login — returns a signed JWT. All subsequent calls carry Authorization: Bearer <token>. Roles: ADMIN, DEVELOPER, VIEWER.

2

Deployment is created — risk assigned automatically

POST /api/deployments — the Risk Assessment Engine assigns HIGH for PRODUCTION, MEDIUM for STAGING, LOW for DEV/TEST. No manual classification needed.

3

Config/metadata changes are linked to the deployment

POST /api/metadata-changes — records what changed (CREATED/MODIFIED/DELETED), which component, old & new values, and who made the change. Linked to the parent deployment by ID.

4

Dashboard shows live risk summary

GET /api/stats — returns aggregate counts by risk level / environment / status and a merged recent-activity timeline of deployments + metadata changes. Powers the React Recharts dashboard.

5

Compliance export — one click, audit-ready CSV

GET /api/export/deployments.csv — streams RFC 4180 CSV with optional ?environment=PRODUCTION&riskLevel=HIGH filters. Attach directly to ISO 27001 / SOC 2 audit packages. Built with Apache Commons CSV.

Example API calls

GET /api/export/deployments.csv
→ All deployments as RFC 4180 CSV
GET /api/export/deployments.csv?environment=PRODUCTION&riskLevel=HIGH
→ Only high-risk production deploys
GET /api/export/metadata-changes.csv?changeType=DELETED
→ All deleted-component audit records

Quick Start

# Clone and start
git clone https://github.com/rayenx2/Blastradius.git
cd Blastradius && cp .env.example .env
docker compose up -d --build
# Open after ~60s:
http://localhost:8050 # React dashboard
http://localhost:8051/audittrail/swagger-ui.html # Swagger UI
http://localhost:8051/audittrail/actuator/health # Health check
# Default: admin_user / AdminPass123!
